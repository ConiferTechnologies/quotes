Project Overview

A comprehensive healthcare provider routing and scheduling optimization system designed to efficiently assign Care Providers (CPs) and Emergency Medical Technicians (EMTs) to patients.

Executive Summary

This project will develop a custom optimization solution using Google OR-Tools to automate the daily assignment of healthcare providers to patients across multiple hospitals and outpatient locations. The system will replace manual scheduling processes with an intelligent, automated solution that considers multiple factors including provider expertise, patient history, geographic proximity, and workload balancing.

Use Cases

Primary and secondary use cases for the optimization system

Primary Use Cases

  • Daily Morning Assignments: Automatically assign CPs to inpatients based on optimized routing and continuity factors
  • Outpatient Scheduling: Efficiently schedule EMTs and CPs for outpatient visits with geographic clustering
  • Tuck-In Visit Assignments: Optimize evening/night visit assignments for patients requiring basic care
  • Dynamic Reassignment: Real-time rescheduling when providers call out or patient conditions change
  • Workload Management: Ensure balanced assignments while respecting maximum visit limits

Secondary Use Cases

  • Performance Analytics: Track provider efficiency and total time spent traveling
  • Schedule Optimization: Identify opportunities to reduce travel time and improve care delivery
  • Resource Planning: Forecast staffing needs based on patient volume and geographic distribution

Requirements

Functional and technical requirements for the optimization system

Core Data Requirements

  • CP Daily Work Location: Daily work locations (Beam, North, etc.), certifications, specializations
  • Patient Data: GPS coordinates, Medical Record Numbers (MRN), admitting hospitals (CMC, Cab, etc.)
  • Care Constraints: EMT-eligible patients, pediatric patients, medication restrictions (Lasix, Insulin), and PT acuity
  • Visit History: Previous provider-patient interactions for continuity scoring

Assignment Constraints

  • Provider Limits: CPs maximum 4 inpatients (8 visits), EMTs/CPs maximum 6 outpatients
  • Scope Restrictions: EMT limitations for specialized care requirements
  • Geographic Boundaries: Multi-hospital system coverage area
  • Scheduling Windows: Morning assignments, tuck-in visits, emergency reassignments

Optimization Criteria

  • Daily PT to CP Match Score: From -2 to 20 based on a static set of rules
  • Continuity Scoring: Previous day (+2), previous stay (+1) bonuses
  • Geographic Efficiency: Distance-based scoring from provider base locations
  • Hospital Clustering: Same-hospital visit bonuses (+5)
  • Travel Optimization: Proximity-based scoring for sequential visits
  • Workload Balancing: Penalty system for overloaded providers

Technical Architecture

  • Backend: Python-based application using Google OR-Tools optimization engine
  • Database: PostgreSQL for patient/provider data storage
  • API Layer: RESTful services for data integration and mobile access
  • Web Interface: React-based dashboard for schedule management
  • Integration: Hospital information systems integration

Security & Compliance

  • HIPAA Compliance: Protected health information handling
  • Data Encryption: At-rest and in-transit encryption
  • Access Controls: Role-based authentication and authorization
  • Audit Logging: Complete action tracking for compliance

Risk Assessment

Potential risks and mitigation strategies

Technical Risks

Integration Complexity

Integration with Hospital IT systems often requires a series of security reviews, approvals, and is subject to final approval by the Hospital CIO. Two possible work arounds are to build the system to a security specification such as SOC2 Type2 or HITRUST. The other option is to manually load the data into the routing application via a CSV download from the EMR or similar process.

Mitigation: Implement SOC2 Type2 compliance or develop manual data loading processes as backup integration methods.

Data Quality

Incomplete or inaccurate source(s) of data will render the system inoperable

Mitigation: Implement comprehensive data validation and quality checks throughout the system.

HIPAA Compliance

PT GPS location, medication, and MRN are all directly identifiable information which would mean this system must meet HIPAA compliance.

Mitigation: Implement comprehensive HIPAA compliance measures including encryption, access controls, and audit logging.

Recommendation

This is a classic logistics and computer science challenge often referred to as the Traveling Salesman Problem (TSP) or the Vehicle Routing Problem (VRP). Every company that manages drivers must solve this problem. As such many commercial off the shelf options exist to solve this problem. While each has limitations each will likely solve the core problem (CP to PT routing) effectively, as such Conifer recommends these options be explored prior to engaging in a software development project.
